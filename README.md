# База аварийных остановов ГПА — веб-версия

Поиск по аварийным остановам, дефектам и работам по их устранению для КС-1 «Алимтау».

## Структура

```
index.html         — сама страница поиска (открывается в браузере)
accidents.json     — база аварий (единственный источник правды для аварий)
defects.json       — база дефектов без остановов
acts/              — все PDF-акты (расследования, дефектные, выполненных работ)
```

Страница загружает данные из `accidents.json` и `defects.json` через `fetch()`.
**Поэтому просто открыть `index.html` двойным кликом не получится** — браузеры
блокируют `fetch()` для локальных файлов (`file://`). Нужен один из двух вариантов:

## Вариант А: разместить на GitHub Pages (рекомендуется)

1. Зайдите на github.com, создайте новый репозиторий (например, `gpa-incidents`).
   Публичный или приватный — оба варианта поддерживают Pages (приватный —
   на платных планах GitHub Pro/Team/Enterprise, если нужен закрытый доступ).
2. Загрузите туда все 4 объекта из этой папки (`index.html`, `accidents.json`,
   `defects.json`, папку `acts/`) — через веб-интерфейс («Add file» → «Upload files»,
   можно перетащить всю папку) или через `git push`.
3. В репозитории откройте **Settings → Pages**.
4. В поле «Source» выберите **Deploy from a branch**, ветку **main**, папку **/ (root)**.
   Сохраните.
5. Через 1–2 минуты GitHub покажет адрес вида:
   `https://<ваш-логин>.github.io/gpa-incidents/`
   Это и есть рабочая ссылка на поиск — можно рассылать коллегам.

Дальше при появлении новой аварии: правите `accidents.json` (или `defects.json`)
прямо на GitHub (кнopка карандаша у файла — «Edit») или заливаете новый файл —
сайт обновится автоматически в течение минуты.

## Вариант B: локальный просмотр без GitHub

Если нужно посмотреть прямо сейчас без загрузки на GitHub — запустите локальный
веб-сервер из этой папки (Python есть почти везде):

```
cd путь/к/этой/папке
python3 -m http.server 8000
```

Затем откройте в браузере `http://localhost:8000`.

## Добавление новой аварии

1. Откройте `accidents.json`, скопируйте последний объект `{...}` как шаблон.
2. Заполните поля (`act`, `date`, `gpa`, `cause`, `measures` и т.д.).
3. Если есть акт устранения — впишите в массив `remediation` внутри записи,
   поле `work_source` должно точно совпадать с именем PDF-файла в папке `acts/`.
4. Положите сам PDF-акт в папку `acts/`.
5. Загрузите обновлённые `accidents.json` и новый PDF на GitHub — готово.

Для самостоятельных дефектов (не привязанных к автоостанову) — та же логика,
но в файле `defects.json`.
